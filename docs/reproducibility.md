# Reproducibility Notes

This repository is designed to support two levels of reproducibility.

## Artifact Inspection

The included compact artifacts allow the reported final-study numbers to be inspected without rerunning live GitHub collection. The most important files are:

- `data/interim/final_sample/final_sample.csv`
- `data/interim/final_sample/collection_completeness_audit.json`
- `data/interim/final_sample/classified_commits/classified_commits_summary.json`
- `data/interim/final_sample/classified_issues/classified_issues_summary.json`
- `data/processed/repo_metrics/final_sample/repository_metrics.csv`
- `results/final_sample/tables/descriptive_summary_by_language.csv`
- `results/final_sample/reporting_summary.json`
- `results/final_sample/final_study_results_checkpoint.json`

## Full Rerun

A full rerun requires a GitHub token and live access to public GitHub repository data. Repository state may change after the original run, so exact reruns may differ if repositories are renamed, deleted, archived, or modified.

Long GitHub-facing stages may hit API rate limits. The enrichment stage supports safe resume behavior:

```powershell
python scripts/collect/enrich_final_candidates.py --resume
```

Downstream final-study stages should use the locked sample manifest:

```powershell
--sample-file data/interim/final_sample/final_sample.csv
```

## Data Included and Excluded

Included:

- final sample and reserve manifests
- candidate-screening and eligibility artifacts
- collection, classification, aggregation, and reporting summaries
- repository-level metric CSV used for the paper
- generated descriptive tables and figures

Excluded:

- large raw commit and issue payloads
- large row-level classified commit and issue CSVs
- private development notes and draft manuscript files
- local environment files and tokens

The excluded raw and row-level artifacts can be regenerated by rerunning the pipeline.
